- [ ] Create `SuccessionPlan` model (enhance existing) + tests

- [ ] Create database indices for performance

    - (organization_id, created_at)

#### B1.2: Talent Risk Analytics Models

- [ ] Create `TalentRiskIndicator` model + migration

### Phase B2: Business Logic Services (Days 6-12, Apr 6-12)

#### B2.1: SuccessionPlanningService

- [ ] Implement `calculateSkillMatch(person, targetRole, scenario)`: void
    - Match person's current skills vs. target role requirements
    - Factor in training in flight, experience, past roles
    - Return skill_match_score (0-100)
    - **Est. LOC:** 180

- [ ] Implement `assessReadiness(successor, targetRole)`: string
    - Evaluate person's readiness level based on:
        - Skill match (40%)
        - Experience in similar roles (30%)
        - Mentor availability (20%)
        - Motivation/interest signals (10%)
    - Return: "junior" / "intermediate" / "senior" / "expert"
    - **Est. LOC:** 140

- [ ] Implement `calculateSuccessionCoverage(scenario)`: array
    - For each role in scenario, count successors by readiness
    - Return: {role_id, covered_count, at_risk_count, critical_count, coverage_pct}
    - **Est. LOC:** 100

**Est. LOC:** 420

#### B2.2: TalentRiskAnalyticsService

- [ ] Implement `analyzeVolatilityRisk(person, organization)`: float
    - Consider: turnover in dept, person's tenure, role criticality, salary relative to market
    - Return risk_score (0-100)
    - **Est. LOC:** 150

- [ ] Implement `predictRetentionRisk(person, scenario)`: float
    - ML-lite: simple weighted scoring based on:
        - Career progression opportunity in scenario (30%)
        - Skill alignment with future roles (25%)
        - Team dynamics & engagement (20%)
        - Compensation competitiveness (15%)
        - Unknown factors buffer (10%)
    - Return retention_probability (0-100)
    - **Est. LOC:** 160

- [ ] Implement `calculateSkillGaps(scenario)`: array
    - For each future role, identify missing skills across workforce
    - Prioritize by role importance + gap severity
    - Return: [{skill, gap_count, priority, training_recommendations}]
    - **Est. LOC:** 140

**Est. LOC:** 450

#### B2.3: TransformationRoadmapService

- [ ] Implement `generateTransformationPlan(scenario, options)`: void
    - Create phased migration plan from current to future state
    - Phase 1 (Months 0-6): Foundation (hire/develop key roles)
    - Phase 2 (Months 6-12): Scale (build teams)
    - Phase 3 (Months 12+): Optimize (refine, exit legacy roles)
    - **Est. LOC:** 250

- [ ] Implement `identifyBlockers(phase)`: array
    - Analyze dependencies between tasks
    - Flag resource conflicts, skill bottlenecks, timeline risks
    - **Est. LOC:** 120

**Est. LOC:** 370

**Subtotal Phase B2:** ~1,240 LOC (business logic)

## 🔄 Parallel Execution Strategy

### Synchronization Points

| Date        | Milestone                   | Action                                                      |
| ----------- | --------------------------- | ----------------------------------------------------------- |
| **Apr 1**   | Sprint Kickoff              | Both branches created from `main`, daily standup starts     |
| **Apr 4-5** | Mid-point check             | Sync Track A + B on API contracts, Test framework alignment |
| **Apr 10**  | Integration Testing Preview | Both tracks preview integration scenarios                   |
| **Apr 15**  | Feature Freeze              | Both features nominally feature-complete, testing focus     |
| **Apr 18**  | Code Review                 | Both branches code-reviewed, conflicts resolved             |
| **Apr 21**  | Merge to Main               | Both feature branches → `develop` → `main` (final tag)      |

### Branch Management

```
main (stable)
├── feature/lms-hardening-v2.0 (Track A)
│   ├── lms/course-builder (Day 1-4)
│   ├── lms/assessment-engine (Day 5-8)
│   ├── lms/learning-analytics (Day 7-12)
│   └── lms/multimedia (Day 13-16)
│
└── feature/scenario-planning-phase2-v2.0 (Track B)
    ├── scenario/models (Day 1-5)
    ├── scenario/services (Day 6-12)
    ├── scenario/api (Day 13-17)
    ├── scenario/policies (Day 18-19)
    └── scenario/tests (Day 20-21)
```
