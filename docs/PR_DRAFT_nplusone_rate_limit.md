PR Draft: N+1 Audit & Rate Limiting Sprint

## Summary

Small-to-medium risk refactor focused on eliminating N+1 query hotspots across reporting and analytics endpoints and consolidating a few executive aggregates into single queries. Also added safer caching inside `ScenarioAnalyticsService` to avoid repeated per-item DB calls.

## Files changed (key)

- app/Services/ScenarioAnalyticsService.php
- app/Services/ImpactReportService.php
- app/Services/TalentRoiService.php
- app/Services/Intelligence/ImpactEngineService.php
- openmemory.md

## What changed

- Prefetching: `ScenarioAnalyticsService::ensureScenarioCache()` now precaches `competency_names` and several aggregates (competency_skills, scenario_role_skills, people_role_skills aggregates, closure strategies, etc.).
- `calculateImpact()` now uses the scenario cache exclusively and avoids per-skill/role fallback DB queries.
- `ImpactReportService` passes loaded `Scenario` models through to avoid duplicate finds, and `getTalentPipelineMetrics()` consolidates three counts into a single SQL.
- `TalentRoiService::getExecutiveAggregates()` was extended to include `avg_turnover_risk` and other aggregates; `getExecutiveSummary()` reuses these aggregates and exposes some for callers to avoid duplicate counts.
- Several call sites (ImpactReportService) now reuse precomputed aggregates (e.g., `upskilled_count`) instead of re-querying.
