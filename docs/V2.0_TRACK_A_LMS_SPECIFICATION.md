## 📋 Overview

Enhance the existing LMS with professional course creation UI, advanced assessment engine, learning analytics dashboard, and multimedia support.

### Goals

1. ✅ Enable non-technical course authors to build rich learning experiences
2. ✅ Provide detailed progress tracking & analytics for students & instructors
3. ✅ Support multimedia-rich content (video, interactive modules)
4. ✅ Mobile-responsive design (375px minimum)
5. ✅ Sub-2-second page loads

### Success Metrics

- **Adoption:** 10+ courses created in UAT
- **Performance:** p95 latency < 1.5s, 90th percentile < 2.0s
- **Quality:** 0 critical bugs, < 3 medium bugs
- **Accessibility:** WCAG 2.1 AA compliance

## 🎯 Phase A1: Course Builder UI (Apr 1-4, Days 1-4)

### A1.1: CourseBuilder.vue Component

**Purpose:** Master interface for creating/editing course structure

**File:** `resources/js/components/LMS/CourseBuilder.vue` (600 LOC)

#### Features

- **Course Metadata Form**
  - Title, description, thumbnail
  - Category, difficulty level (beginner/intermediate/advanced/expert)
  - Duration estimate (hours)
  - Prerequisites (multi-select from other courses)
  - Tags (skill-based: React, Python, Leadership, etc.)

- **Lesson Tree Editor**
  - Drag-and-drop lesson reordering (vue-tree-dragdrop)
  - Add/edit/delete lessons
  - Lesson nesting (modules > lessons > sub-lessons, 3 levels max)
  - Each lesson has: title, description, duration, learning objectives (JSON), prerequisites

- **Publishing Workflow**
  - Draft/Published/Archived status
  - Publish date scheduling
  - Visibility toggles (Public/Private/Cohort-specific)

#### API Calls

- `POST /api/lms/courses` - Create new course
- `PATCH /api/lms/courses/{id}` - Update course metadata
- `GET /api/lms/courses/{id}/structure` - Fetch lesson hierarchy
- `POST /api/lms/courses/{id}/lessons` - Add lesson
- `PATCH /api/lms/courses/{id}/lessons/{lessonId}` - Update lesson
- `DELETE /api/lms/courses/{id}/lessons/{lessonId}` - Delete lesson
- `POST /api/lms/courses/{id}/publish` - Publish course

#### Components Used

- Vuetify v3: VCard, VForm, VTextarea, VChip, VDatePicker
- Tailwind: Flex, Grid layouts
- TipTap: Rich text editor for descriptions
- draggable: Lesson reordering

- [ ] Render course form with all fields ✅
- [ ] Add/remove lessons from tree ✅
- [ ] Drag reordering updates order ✅
- [ ] Publish toggles course status ✅
- [ ] Validation: required fields, title length limits ✅

## 🎯 Phase A2: Assessment Engine (Apr 5-10, Days 5-10)

### A2.1: AssessmentBuilder.vue Component

**Purpose:** Create quizzes, tests, and assignments

**File:** `resources/js/components/LMS/AssessmentBuilder.vue` (450 LOC)

#### Features

- **Assessment Metadata**
  - Title, description, duration limit (minutes, 0 = unlimited)
  - Passing score (%)
  - Passing criteria: "Any" / "All correct" / "Score threshold"
  - Retake policy: allowed/limited (N) / one-time
  - Show answers immediately / after submission / never
  - Question shuffling (toggle)

- **Question Builder (Drag-drop addable)**
  - **Multiple Choice (MCQ)**
    - Question text + image
    - 2-5 options, radio select (single correct)
    - Points per question (default 1)
    - Explanation for each answer

  - **Multiple Select**
    - 2-5 options, checkboxes (1+ correct)
    - Points per question

  - **Short Answer**
    - Expected answer(s) (free text)
    - Case-insensitive matching
    - Partial credit rules (if implemented)

  - **Coding Challenge** (Future - scope for v2.0.1)
    - Template code + test cases
    - Language support

- **Scoring Configuration**
  - Total points calculator (auto-sum)
  - Percentage to letter grade mapping (A/B/C/D/F)

#### API Calls

- `POST /api/lms/assessments` - Create new assessment
- `PATCH /api/lms/assessments/{id}` - Update assessment metadata
- `GET /api/lms/assessments/{id}/questions` - Fetch all questions
- `POST /api/lms/assessments/{id}/questions` - Add question
- `PATCH /api/lms/assessments/{id}/questions/{qId}` - Update question
- `DELETE /api/lms/assessments/{id}/questions/{qId}` - Delete question

#### Subcomponents

- `QuestionMCQBuilder.vue` (120 LOC) - MCQ template
- `QuestionShortAnswerBuilder.vue` (80 LOC) - Text answer template
- `ScoringConfig.vue` (100 LOC) - Points & grading rules

#### Tests

- [ ] Add/edit/delete questions ✅
- [ ] Scoring auto-calculates correctly ✅
- [ ] Validation: required fields, points range ✅

### A4.1: VideoPlayer.vue Component + HLS Integration

**Purpose:** Adaptive video streaming with progress tracking

**File:** `resources/js/components/LMS/VideoPlayer.vue` (250 LOC)

#### Features

- **Video Playback**
  - HLS.js integration for adaptive bitrate streaming
  - Play/pause/seek controls
  - Volume control
  - Fullscreen mode
  - Playback speed menu (0.75x, 1x, 1.25x, 1.5x, 2x)

- **Captions**
  - Display subtitle/caption file (VTT format)
  - Toggle captions on/off
  - Font size/style settings

- **Progress Tracking**
  - Auto-report watch time to backend
  - Mark as "viewed" if watched 90%+
  - Prevent rewinding past "last completed position" (for structured learning)

- **Interactive Overlays** (Future v2.0.1)

#### Props

- `videoUrl` - HLS stream URL (m3u8)

- `duration` - Lesson duration

#### API Calls

- `POST /api/lms/lessons/{id}/video-watch-time` - Report time watched
- `PATCH /api/lms/lessons/{id}/mark-viewed` - Mark as viewed

#### Dependencies

- `hls.js` - HLS streaming library (npm package)
- `video.js` - Video player (optional, or use native HTML5 <video>)

**File:** `resources/js/components/LMS/InteractiveModule.vue` (200 LOC)

#### Features

- **IFrame Sandboxing**
  - Allow-scripts, allow-same-origin, allow-popups (minimal perms)
  - Response to xAPI events (IFrame posts messages)

- **Interaction Tracking**
  - Listen for xAPI track events (Content completion, questions answered, etc.)

- `lessonId` - For activity tracking

#### API Calls
